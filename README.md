# Aleph Studio Website

Статический сайт Aleph Studio, опубликованный через GitHub Pages.

Сейчас проект собран как один большой HTML-файл со встроенными стилями, скриптами, данными сайта и скрытой админ-панелью. Это рабочий вариант для быстрого деплоя, но его уже имеет смысл разнести на нормальную структуру, чтобы дальше было проще поддерживать код.

## Что умеет сайт

- Показывает главные продукты, каталог, команду и социальные ссылки.
- Имеет скрытую админ-панель для редактирования контента прямо в браузере.
- Поддерживает локальное сохранение черновика через `localStorage`.
- Поддерживает публикацию для всех через GitHub API по токену.
- Умеет загружать файл продукта в репозиторий при общем сохранении.

## Текущая структура

```text
web/
  .nojekyll
  index.html
  website.html
  README.md
  REFACTOR_PROMPT.md
```

## Важные файлы

- `index.html` — основной файл для GitHub Pages.
- `website.html` — рабочая копия того же сайта; сейчас файл дублирует `index.html`.
- `.nojekyll` — отключает Jekyll-обработку на GitHub Pages.
- `README.md` — документация по текущему состоянию проекта.
- `REFACTOR_PROMPT.md` — готовый промт для следующего чата, если нужно разнести монолит на нормальную структуру.

## Как сейчас устроен проект

- Весь интерфейс, стили, логика и данные лежат внутри одного HTML-файла.
- Контент сайта хранится во встроенном JS-блоке `DEFAULT_SITE_DATA`.
- Админка редактирует общий черновик, который потом можно:
  - применить как preview,
  - сохранить локально,
  - сохранить в GitHub для всех пользователей.
- При публикации в GitHub HTML обновляется через Contents API.
- Если в товаре выбран файл, админка может отправить его в репозиторий тем же токеном.

## GitHub Pages

Сайт рассчитан на публикацию через GitHub Pages из ветки `main`, из корня репозитория.

## Сохранение

### Локально

- Сохраняет черновик в браузере через `localStorage`.
- Изменения видны только на этом устройстве и в этом браузере.

### Для всех

- Требует GitHub token.
- Обновляет опубликованный HTML через GitHub API.
- Может одновременно загрузить файл товара в репозиторий.

Для токена нужен доступ как минимум:

```text
Contents: Read and write
```

## Почему проект стоит разнести

Сейчас сайт уже работает, но монолитный HTML неудобен для дальнейшей разработки:

- слишком большой файл;
- сложно поддерживать стили;
- сложно править админку отдельно от публичной части;
- трудно переиспользовать код;
- неудобно добавлять новые секции, данные и загрузки файлов;
- высок риск случайно сломать несвязанные части страницы.

## Желаемая цель следующего этапа

Разнести проект на нормальную структуру примерно такого вида:

```text
web/
  index.html
  assets/
    css/
    js/
    icons/
    files/
  src/
    data/
    components/
    admin/
    github/
```

При этом важно сохранить:

- текущий внешний вид;
- текущую логику админки;
- локальное сохранение;
- общее сохранение через GitHub token;
- загрузку файлов товара;
- совместимость с GitHub Pages.
